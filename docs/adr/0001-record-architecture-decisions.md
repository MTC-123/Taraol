# ADR 0001: Record architecture decisions

## Status

Accepted

## Context

This project has independent demo, detection, and MCP layers. Decisions that affect their
boundaries must remain easy to discover as the hackathon implementation evolves.

## Decision

Record consequential architecture decisions as short Architecture Decision Records in
`docs/adr/`, numbered sequentially.

## Consequences

Contributors can understand why a boundary or operational choice exists without relying on
chat history.
